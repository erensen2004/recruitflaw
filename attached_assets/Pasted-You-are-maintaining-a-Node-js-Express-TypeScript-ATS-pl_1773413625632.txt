# 1. Enforce Object-Level Authorization for Private Storage

Currently the route:

GET /storage/objects/*path

only requires authentication, which is **not sufficient**.

Fix this so that **file access is validated against object ownership**.

### Requirements

1. Before serving a private object:

   * Resolve the object path
   * Verify access using:

   objectStorageService.canAccessObjectEntity(...)

2. If access is not allowed:

   return 403 Forbidden

3. Never allow authenticated users to access objects belonging to other organizations.

### Expected Logic

* Admin → full access
* Client → only objects belonging to their company
* Vendor → only objects belonging to their vendor submissions

If object metadata is missing or invalid → return 403.
